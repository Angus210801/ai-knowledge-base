## LLaMA系列模型演进

### LLaMA 1（2023年2月24日）
- **参数**：7B / 13B / 33B / 65B
- **许可**：仅限研究用途
- **意义**：
  - 证明了小模型经过充分训练可以匹敌大模型
  - 65B模型性能接近GPT-3.5
- **影响**：模型权重被泄露，催生了整个开源LLM生态
  - Alpaca（Stanford）
  - Vicuna
  - Guanaco
  - 数百个微调版本

### LLaMA 2（2023年7月18日）
- **参数**：7B / 13B / 70B
- **许可**：开源商用许可
- **意义**：
  - 首个允许商用的高质量开源大模型
  - 与Microsoft合作分发
- **训练数据**：2万亿token
- **特点**：
  - 上下文4K
  - 针对对话场景优化（LLaMA 2-Chat）

### Code LLaMA（2023年8月）
- 专门针对编程优化的版本
- 支持多种编程语言
- 参数：7B / 13B / 34B / 70B

### LLaMA 3（2024年4月18日）
- **参数**：8B / 70B（后推出405B）
- **训练数据**：15万亿+ token
- **上下文**：8K
- **关键提升**：
  - 多语言能力大幅提升
  - 推理能力显著增强
  - 8B模型性能超越之前的70B
- **405B版本**（2024年7月）：
  - 当时最大的开源模型
  - 性能接近GPT-4

### LLaMA 3.1（2024年7月23日）
- **参数**：8B / 70B / 405B
- **上下文**：128K
- **特点**：
  - 超长上下文
  - 多语言支持增强
  - 工具调用能力

### LLaMA 3.2（2024年9月25日）
- **新增**：
  - 多模态版本（11B/90B Vision）
  - 轻量级模型（1B/3B）
- **特点**：
  - 1B和3B可在手机上运行
  - Vision版本支持图像理解

### LLaMA 3.3（2024年12月）
- 70B模型性能接近之前的405B
- 更高效，成本更低

### LLaMA 4（2025年4月5日）

#### LLaMA 4 Scout
- **参数**：109B（17B激活，16专家MoE）
- **上下文**：1000万token！（行业最长）
- **特点**：可以在单个H100 GPU上运行

#### LLaMA 4 Maverick
- **参数**：400B（17B激活，128专家MoE）
- **上下文**：100万token
- **特点**：性能媲美GPT-4o和Gemini 2.0

#### LLaMA 4 Behemoth（预览）
- **参数**：2万亿
- **状态**：训练中，尚未发布

## 核心技术特点

### 开源策略
- 完整的模型权重开放下载
- 允许商用（LLaMA 2起）
- 催生了庞大的开源生态
- **意义**：让每个人都能使用和改进大模型

### 高效训练
- 证明了"数据质量 > 模型规模"
- 小模型经过充分训练可以匹敌大模型
- 8B模型在很多场景下够用

### MoE架构（LLaMA 4）
- Mixture of Experts（混合专家）
- 总参数大，但每次推理只激活部分参数
- 效率更高，成本更低

## LLaMA的影响力

### 开源生态
LLaMA的发布催生了：
- **Alpaca**：Stanford的轻量级微调
- **Vicuna**：GPT-4级别对话能力
- **Ollama**：本地运行LLaMA的工具
- **llama.cpp**：C++推理引擎
- **Hugging Face**生态的繁荣

### 行业影响
- 证明了开源模型可以与闭源模型竞争
- 降低了AI应用的门槛
- 推动了本地部署和隐私保护
